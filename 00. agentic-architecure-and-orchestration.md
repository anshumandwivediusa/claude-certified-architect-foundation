# Domain 1: Agentic Architecture & Orchestration (27%)

## Agentic Architecture and Execution Models

Agentic architecture is the foundation of modern AI systems that can independently plan, reason, and execute complex tasks. Unlike traditional chatbots that simply generate a single response to each prompt, agentic systems operate as autonomous problem solvers capable of making decisions, invoking tools, evaluating results, and adapting their strategy until an objective is achieved.

In Claude Code, an agent is not just a language model—it is an orchestrated system that combines reasoning, planning, tool execution, memory, and iterative feedback into a continuous execution loop. Rather than responding once and stopping, the agent repeatedly observes the current state, determines the next best action, executes that action, evaluates the outcome, and continues until the task is completed or requires human intervention.

An execution model defines **how** this reasoning process is organized. It specifies how tasks are decomposed into smaller steps, how tools are selected and invoked, how intermediate results are validated, how context is preserved across iterations, and how the system determines when to continue, retry, delegate work to specialized agents, or terminate execution. Different execution models balance autonomy, reliability, cost, and human oversight depending on the complexity of the problem.

Understanding agentic architecture requires more than knowing what an AI model can generate. It involves understanding the interaction between the reasoning engine, execution loop, tool ecosystem, memory management, permission enforcement, and coordination mechanisms that enable an autonomous software engineering workflow.

For the Claude Code Architect Foundation exam, candidates should understand:

* The characteristics of an AI agent and how it differs from a traditional chatbot or workflow.
* The architecture of iterative reasoning and execution loops.
* The role of planning, observation, action, and evaluation during task execution.
* How execution models determine task decomposition, orchestration, and completion.
* The relationship between the agent, tools, memory, permissions, and human oversight.
* When autonomous execution is appropriate versus when human approval or intervention is required.

A solid understanding of agentic architecture and execution models provides the conceptual foundation for every other topic in the certification, including multi-agent systems, tool orchestration, context management, reliability, and governance.
