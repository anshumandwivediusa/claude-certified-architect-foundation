# Claude Code Agentic Loop

## Agentic Architecture Context
- **Agentic AI** → Autonomous, Goal Oriented, Dynamic Planning and Reasoning, Systems that Goal → Plan → Reason → Act → Evaluate → Adapt → Repeat → Complete.  
- **Difference from chatbots** → not just one-shot responses; agents iterate until a goal is achieved.  
- **Claude Code** → implements this via orchestration: reasoning engine + execution loop + tools + memory.

## Example : Single Method

```
import anthropic

client = anthropic.Anthropic()

# Define banking tool
tools = [
    {
        "name": "validate_customer",
        "description": "Validate customer account and KYC status",
        "input_schema": {
            "type": "object",
            "properties": {
                "customer_id": {"type": "string"},
                "account_number": {"type": "string"}
            },
            "required": ["customer_id", "account_number"],
        },
    }
]

# Tool execution logic
def execute_tool(name, tool_input):
    if name == "validate_customer":
        # Simulated validation logic
        if tool_input["customer_id"] == "CUST123" and tool_input["account_number"] == "ACC456":
            return "Customer is valid: KYC complete, account active, no fraud flags."
        else:
            return "Customer validation failed: account inactive or KYC pending."
    return "unknown tool"

# Initial user request
messages = [{"role": "user", "content": "Validate customer CUST123 with account ACC456"}]

# Agentic execution loop
while True:
    response = client.messages.create(
        model="claude-sonnet-5",
        max_tokens=1024,
        tools=tools,
        messages=messages,
    )

    messages.append({"role": "assistant", "content": response.content})

    match response.stop_reason:

        case "tool_use":
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    result = execute_tool(block.name, block.input)
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": result,
                    })
            messages.append({"role": "user", "content": tool_results})
            continue

        case "pause_turn":
            continue

        case "end_turn" | "stop_sequence":
            final_text = "".join(b.text for b in response.content if b.type == "text")
            print(final_text)
            break

        case "max_tokens":
            print("Truncated — consider raising max_tokens or summarizing history.")
            break

        case "refusal":
            print("Refused:", response.stop_reason, getattr(response, "stop_details", None))
            messages = messages[:1]
            break

        case "model_context_window_exceeded":
            print("Hit context window — trim or summarize `messages` and retry.")
            break

```

## Code Components
- **Anthropic client** → entry point to interact with Claude (`client = anthropic.Anthropic()`).
- **Tools** → external functions the agent can call.  
  - Example: `get_weather` with schema validation.  
  - Tools extend the agent beyond its parameters.
- **Execution function** → `execute_tool` runs the tool logic and returns results.  
- **Messages** → conversation state; appends user + assistant turns for memory.


## Execution Loop
- **Continuous cycle (`while True`)** → agent keeps reasoning until termination.  
- **Response handling (`stop_reason`)** → defines agent’s next step:
  - **tool_use** → agent invokes external tool, returns result, loops again.  
  - **pause_turn** → server-side tool loop reached cap; resend without new input.  
  - **end_turn / stop_sequence** → agent outputs final text and halts.  
  - **max_tokens** → context too long; requires summarization or higher token limit.  
  - **refusal** → policy block; reset context.  
  - **model_context_window_exceeded** → memory overflow; trim or summarize.


## Agentic Concepts in Action
- **Reasoning engine** → Claude decides whether to generate text or call a tool.  
- **Planning** → decomposes “weather + haiku” into subtasks.  
- **Tool invocation** → executes `get_weather`.  
- **Validation** → integrates tool results back into conversation.  
- **Memory** → `messages` list preserves context across iterations.  
- **Feedback loop** → agent adapts strategy until completion.  
- **Termination conditions** → safe exit points ensure reliability.


## Design Principles Reflected
- **Separation of reasoning & execution** → Claude decides, Python executes.  
- **Explicit state** → `messages` maintains structured context.  
- **Fail safely** → handles refusals, truncations, context overflow.  
- **Iterative optimization** → loop continues until success or stop condition.  
- **Controlled autonomy** → agent acts independently but within defined guardrails.


## Key Takeaways for Exams
- **Agent vs Chatbot** → agent = iterative, tool-using, adaptive; chatbot = one-shot.  
- **Execution loop** → central to agentic design; enables persistence and adaptation.  
- **Tools** → extend capabilities; schema ensures safe structured input.  
- **Stop reasons** → critical for reliability, safety, and controlled autonomy.  
- **Claude Code** → practical example of agentic architecture applied to software engineering workflows.
